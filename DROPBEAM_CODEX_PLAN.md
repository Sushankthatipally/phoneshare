## ⚠️ ARCHITECTURE CHANGE FROM PREVIOUS PLAN

The following are REMOVED entirely:
- ❌ `apps/iphone-web/` — Safari PWA is gone
- ❌ `web_server.rs` — no static file server needed
- ❌ `tokio-tungstenite` WebSocket bridge — no browser clients
- ❌ `jsQR` / `getUserMedia` browser QR scanning
- ❌ `WebCrypto API` — replaced by native crypto on all platforms
- ❌ Any reference to "served from desktop" iPhone flow

The following are ADDED:
- ✅ `apps/ios/` — React Native iOS native app (same codebase as Android)
- ✅ `apps/mobile/` becomes shared Android + iOS via React Native
- ✅ Native crypto on mobile: `react-native-quick-crypto`
- ✅ Native QR scanner: `react-native-vision-camera`
- ✅ Native TCP socket on mobile: `react-native-tcp-socket`
- ✅ Native hotspot control on Android: `WifiManager` via native module

## 🔌 Wire Protocol (Unchanged — Both Platforms Must Match Exactly)

### Packet Header (10 bytes fixed)
```
[1 byte : mode]           0x01=USB  0x02=WiFi  0x03=Hotspot
[1 byte : type]           0x01=handshake  0x02=meta  0x03=chunk
                          0x04=ack  0x05=done  0x06=pin  0x07=reject
[4 bytes: session_id]     random u32, generated by host at session start
[4 bytes: payload_length] u32 big-endian, length of bytes that follow
[N bytes: payload]        encrypted after handshake completes
```

### Handshake Payload (type=0x01, plaintext)
```json
{
  "version": 1,
  "device_name": "Sushank's MacBook",
  "device_type": "desktop",       // "desktop" | "android" | "ios"
  "pub_key": "<base64 ECDH P-256 public key>",
  "session_id": 3141592653
}
```

### PIN Payload (type=0x06, plaintext — sent before encryption established)
```json
{
  "pin": "359741"
}
```

### File Meta Payload (type=0x02, encrypted)
```json
{
  "file_id": "uuid-v4",
  "name": "vacation.jpg",
  "size": 2400000,
  "mime": "image/jpeg",
  "chunks_total": 3,
  "chunk_size": 1048576
}
```

### Chunk Payload (type=0x03, encrypted)
```
[4 bytes: file_id hash]
[4 bytes: chunk_index]
[4 bytes: chunk_size]
[12 bytes: GCM IV]
[N bytes: encrypted data]
[16 bytes: GCM auth tag]
```

### Adaptive Chunk Sizes
| Mode | Chunk Size |
|---|---|
| USB 3.0 | 4 MB |
| WiFi 6 (5 GHz) | 1 MB |
| WiFi 5 (2.4 GHz) | 512 KB |
| Mobile Hotspot | 256 KB |

## 🔐 Security Flow (Same for All Scenarios)

```
HOST                                    JOINER
  │                                        │
  │── generates ECDH P-256 keypair         │
  │── generates session_id (random u32)    │
  │── starts TCP listener on PORT          │
  │                                        │
  │   (QR / mDNS / USB carries IP+PORT     │
  │    + host pubKey + session_id)         │
  │                                        │
  │◀──────── TCP connect ─────────────────│
  │◀──────── HANDSHAKE packet ────────────│── contains joiner pubKey
  │                                        │
  │── derives shared secret (ECDH)         │── derives shared secret (ECDH)
  │── derives AES-256-GCM key (HKDF)       │── derives AES-256-GCM key (HKDF)
  │                                        │
  │── generates 6-digit PIN                │
  │── displays PIN on screen               │
  │                                        │
  │◀──────── PIN packet ──────────────────│── user types PIN
  │                                        │
  │── verifies PIN                         │
  │── sends ACK (encrypted)                │
  │──────── ACK ─────────────────────────▶│
  │                                        │
  │   ✅ SESSION ESTABLISHED               │
  │   All subsequent packets encrypted     │
```
