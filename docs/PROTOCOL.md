# DropBeam Protocol Draft

This repo now includes the first shared protocol package at `packages/protocol`.

## Packet header

The documented packet model follows the original plan:

```text
[1 byte mode]
[1 byte type]
[4 bytes session_id]
[4 bytes payload_length]
[n bytes payload]
```

Current code models this with:

- `TransportMode`
- `PacketType`
- `PacketHeader`
- `ChunkEnvelope`
- `SessionSnapshot`

## Chunk sizing

The current contract exposes `resolveChunkSize(mode)` with these defaults:

- `usb`: `4 MB`
- `wifi`: `1 MB`
- `hotspot`: `256 KB`

These values match the plan well enough for UI and backend integration work.

## What is implemented today

- Shared TypeScript contracts for desktop and iPhone shells
- Packet type labels and numeric codes
- Session, device, and queue data models

## What is still pending

- Binary encoding and decoding
- Streaming chunk IO
- Backpressure and ACK handling
- Resume support
- Real session identifiers generated by the backend
